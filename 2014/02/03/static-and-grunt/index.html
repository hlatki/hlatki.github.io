Static and Grunt

In my last post, I talked about why I've started using using Static as my static site generator. I mentioned that its simplicity is one of the reasons that I like it so much. Static doesn't compress and optimize your CSS -- that's on you. So in this post I'll discuss how and why I used Grunt to slim down my CSS.

Now you might be thinking, "Wait a second, if you want all this fancy minification why didn't you just stick with Octopress?" and I'd be lying if I said this very same question hadn't crossed my mind while I was trying to get everything working the way I wanted. However, I'm sticking with Static and Grunt for several reasons. First, Grunt is an awesome tool that makes front-end development so much better, and learning to use it effectively is something that I think will pay off on many other projects in the future, whereas investing even more time in learning the intricacies of a blogging system seems much less useful/fun. Second, I don't really think that generating a static site and minifying CSS really need to be in the same codebase. I'm totally OK with having one program to convert my markdown and templates into some HTML and then using another tool to manipulate the CSS/JS/whatever files.

I used Yeoman to build the original, blog-less, iteration of hannah.io, and while it was really cool, I hadn't learned much about Grunt. So I decided to use this as an opportunity to dive in and try to learn a thing or to.

Since I used Bootstrap and Typeplate there's a lot of unneeded CSS hanging around. unCSS, and the unCSS grunt task solve this problem. This blog post was a great starting point and had a good discussion of why you might like to cut down on the bloat.
